# アプリ

アプリ用APIの導線のみ。

ネイティブで動かす方法は、`capacitor-test`参照。

## API

Webとは別のアプリ用のAPIを用意してます。

ベースURI /api/appli

## 導入パターン

### Webのみ

コストが最小。

### 最初からアプリも

コストが大きい。

### 途中からアプリ

最初はWebのみで、仕様が固まった後に、マーケティングの観点からのアプリ化。

一番無難そうな流れ。

1. Webで仕様が固まり変更が減る。
1. 拡大したらcapacitorで、低コストなアプリ化
1. さらに拡大したらflutterで、高コストなアプリ化
